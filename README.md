## Project structure
carbench-mx/
│
├── data/
│   ├── raw/          # Original dataset downloaded from Kaggle. Never modified.
│   └── processed/    # Cleaned dataset ready for training. Generated by features.py
│
├── notebooks/
│   └── 01_exploration.ipynb   # Initial analysis: distributions, correlations, nulls
│
├── src/
│   ├── features.py   # Data cleaning and new variable creation (feature engineering)
│   ├── train.py      # Trains the model and saves it to models/
│   └── predict.py    # Reusable function to make predictions
│
├── api/
│   ├── schemas.py    # Defines the input/output format of the API (using Pydantic)
│   └── main.py       # The API itself, built with FastAPI
│
├── models/           # Trained model saved here (.joblib). Not tracked by Git.
│
├── tests/
│   └── test_predict.py   # Automated tests to verify the model works correctly
│
├── requirements.txt  # All libraries needed to run the project
├── Dockerfile        # Instructions to package the app in a Docker container
├── docker-compose.yml # Launches the whole system with a single command
└── README.md         # This file
